# 智能个人助理 Agent 开发项目

## 项目代号
**智助 (SmartAssistant)**

## 项目目标
通过从0到1构建一个基于 LangChain + Python + 智谱免费大模型API的智能助理Agent，系统掌握 Agent 开发的核心技术栈：工具调用、记忆管理、任务规划、RAG知识增强、多步推理与异常处理。

## 技术栈
- 语言：Python 3.10+
- 框架：LangChain、LangChain-Community
- 模型：智谱AI GLM-4-Flash (免费)
- 工具集成：免费API（天气、新闻）、本地SQLite待办、计算器、时间日期
- 记忆：短期对话缓冲记忆 + 长期用户偏好（JSON存储）
- 可选增强：Streamlit UI、向量数据库RAG

## 核心功能
1. 多轮对话理解与回复
2. 工具自主选择与调用：天气查询、新闻摘要、待办增删查改、计算器、时间日期
3. 短期记忆：记住多轮对话上下文
4. 长期记忆：记住用户偏好（如默认城市）
5. 简单多步任务规划：如“查询明天天气，如果有雨提醒我带伞并添加买伞到待办”
6. 安全与错误处理：工具调用异常回退、输出格式校验

## 协作规则
- 所有任务按阶段推进，必须上一阶段验收通过才进入下一阶段。
- 每完成一个子任务，请在对话中汇报进度，我会更新排期状态并给出下一指令。
- 任何新想法或调整，我会随时追加到排期中并同步给你。
- 全程留痕，最终交付完整的项目代码与记录。

## 项目排期与任务分解

### 第0阶段：环境与基础调用验证
**目标**：跑通智谱API，熟悉LangChain基本链与消息格式。

- [ ] 0.1 安装虚拟环境，安装 langchain, langchain-community, zhipuai, python-dotenv
- [ ] 0.2 获取智谱免费API Key，配置环境变量
- [ ] 0.3 使用 LangChain 的 ChatZhipuAI 实现一次简单问答，验证调用成功
- [ ] 0.4 体验 LCEL 链式调用：prompt模板 + model + 输出解析器
- [ ] 0.5 输出第一次验证结果截图或日志，并汇报完成

**验收标准**：可以流式或非流式获得模型回复，无报错。

---

### 第1阶段：Agent核心最小闭环（单工具）
**目标**：掌握 Agent 基本概念，实现“LLM自主选择并调用一个工具”。

- [ ] 1.1 定义第一个工具：计算器（使用LangChain的 @tool 装饰器）
- [ ] 1.2 使用 create_tool_calling_agent + AgentExecutor 创建 Agent
- [ ] 1.3 在循环中调用 Agent，实现单轮工具调用并返回结果
- [ ] 1.4 测试用例：“123 * 456 等于多少”，验证模型能正确调用计算器并返回答案
- [ ] 1.5 添加工具调用日志，观察 Thought-Action-Observation 流程

**验收标准**：Agent 能自动识别需要计算器并返回正确计算结果。

---

### 第2阶段：多工具集成与选择
**目标**：扩展工具集，让 Agent 根据意图自主选择合适工具。

- [ ] 2.1 开发天气查询工具：使用 wttr.in 免费API（无需注册）
- [ ] 2.2 开发时间日期工具：支持“现在几点”、“今天日期”、“下周五日期”等
- [ ] 2.3 将计算器、天气、时间三个工具同时绑定给 Agent
- [ ] 2.4 测试交叉场景：“北京今天天气如何，湿度乘以2是多少” 验证工具选择与串行调用
- [ ] 2.5 处理工具调用异常：网络错误或参数缺失时给出友好回复

**验收标准**：Agent 能在多个工具中准确选择，单次对话支持连续调用多个不同工具。

---

### 第3阶段：对话记忆与多轮交互
**目标**：引入记忆组件，实现带上下文的多轮对话。

- [ ] 3.1 使用 ConversationBufferMemory 或 RunnableWithMessageHistory 实现短期记忆
- [ ] 3.2 实现 session 机制，每个会话独立记忆
- [ ] 3.3 测试多轮对话：“我叫小明”，“我叫什么？”，“帮我查北京天气”，“我刚才问的什么？” 
- [ ] 3.4 观察记忆窗口增长，讨论总结记忆的必要性（为后续扩展埋点）

**验收标准**：Agent 能记住至少前5轮对话内容并正确引用。

---

### 第4阶段：长期记忆与用户偏好
**目标**：引入简单的长期记忆存储（JSON文件），使 Agent 个性化。

- [ ] 4.1 设计用户偏好结构：默认城市、新闻偏好类别等
- [ ] 4.2 开发偏好读写工具：save_user_pref, get_user_pref
- [ ] 4.3 工具注入到 Agent，让 Agent 自主决定何时保存/读取偏好
- [ ] 4.4 测试：“记住我默认城市是上海”，“明天天气” 应自动查询上海（无需再次指定）
- [ ] 4.5 实现偏好持久化到本地JSON，重启后仍有效

**验收标准**：Agent 可以记住并应用用户偏好，重启后记忆不丢失。

---

### 第5阶段：待办事项管理工具与多步规划
**目标**：加入数据库操作，实现真正的多步任务分解与执行。

- [ ] 5.1 使用 SQLite 建表 todos，包含 id、content、status、created_at
- [ ] 5.2 开发待办工具：add_todo, list_todos, complete_todo, delete_todo
- [ ] 5.3 将所有工具（计算器、天气、时间、偏好、待办）集成
- [ ] 5.4 测试复杂多步指令：“如果明天北京有雨，就把买伞加入待办，并提醒我带伞，最后告诉我待办列表”
- [ ] 5.5 分析 Agent 规划路径，优化工具描述以减少选择错误

**验收标准**：Agent 能完成包含条件判断、工具链调用、最终结果汇总的3步以上任务。

---

### 第6阶段：新闻摘要与RAG增强（可选进阶）
**目标**：接入外部信息源，尝试RAG概念。

- [ ] 6.1 开发新闻查询工具：调用 newsapi.org 免费额度 或 搜索模拟
- [ ] 6.2 要求 Agent 对新闻做一句话总结再输出（利用模型总结能力）
- [ ] 6.3 引入简单 RAG：加载本地文本文件到向量库（Chroma），开发检索工具
- [ ] 6.4 实现“根据文档回答”能力，测试知识库问答
- [ ] 6.5 最终整合演示：天气+新闻+待办+知识库+偏好记忆全链路

**验收标准**：Agent 能查询新闻、检索本地文档并给出有依据的回答。

---

### 第7阶段：交互优化与交付
**目标**：打磨产品，输出文档。

- [ ] 7.1 构建简单的 Streamlit 或命令行交互界面
- [ ] 7.2 添加错误边界与日志系统
- [ ] 7.3 编写 README 和项目总结文档
- [ ] 7.4 录制演示视频或截取主要功能动图
- [ ] 7.5 最终验收，归档全部代码与文档

**验收标准**：他人可按README部署运行，所有核心功能正常。
